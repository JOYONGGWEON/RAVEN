> 이 파일은 claude.ai 웹 채팅에서 나눈 기획/설계 논의를 정리한 문서입니다.

## 1. 프로젝트 개요

- **이름**: RAVEN (AI Stock Manager) — 국내/해외 주식 트레이딩 분석 웹앱
- **사용자 수준**: 코딩 초심자. 설명은 쉽게, 전문용어는 풀어서.
- **목표**: ChatGPT로 만든 기존 정적 웹앱(HTML/CSS/JS)을 개선 → 최종적으로는 앱(App) 형태까지 발전
- **핵심 철학**: "뉴스에 낚이는 앱"이 아니라 **실제 거래량, 수급(프로그램/신용/대차/공매도), 캔들·이평선 등 차트 움직임을 기반**으로 판단하는, 월가/전문 트레이더 방식의 분석 도구를 지향함

## 2. 기존 코드 현황 (v6.6 / v7.0, GitHub Pages 배포 중)

파일 구성: `index.html`, `style.css`(1447줄), `app.js`(2427줄)

### 현재 구조
- **화면 흐름**: PIN 입력(`0524`) → "GOOD DAY SIR" 인트로 → 티커 입력 → 분석 결과 화면
- **데이터 소스**: Yahoo Finance API를 `corsproxy.io`(무료 공개 프록시)를 통해 호출 — **완전한 클라이언트 사이드(백엔드 없음)**
- **매크로 대시보드**: 미국 10년물 금리, VIX, USD/KRW 환율, BTC
- **기술적 분석 엔진**: EMA(5/20/60/120), RSI(Wilder), MACD, 스윙 포인트 클러스터링 기반 지지/저항선, 캔들 패턴 12종 자동 인식, R:R 계산, 목표가/손절가, 포지션 사이즈 계산기
- **점수 시스템**: 여러 지표를 규칙 기반(if-else)으로 합산해 0~99점 + S/A/B/C/D 랭크 부여 (⚠️ "AI"라는 이름이지만 실제로는 진짜 LLM이 아니라 규칙 기반 로직임)
- **차트**: TradingView 무료 위젯 임베드
- **전략 요약**: 계산된 지표를 조합한 규칙 기반 자연어 코멘트 생성

### 발견된 주요 문제점 (개선 필요)
1. **PIN코드가 app.js에 평문으로 노출** — 개발자도구로 누구나 확인 가능. 보안 개선 필요 (서버 인증으로 이전)
2. **`corsproxy.io` 공개 프록시 의존** — 트래픽 제한/다운타임 위험. 자체 서버 구축 후 완전히 제거해야 함
3. **국내 주식(KOSPI/KOSDAQ) 미지원** — Yahoo Finance 티커만 가능. 원화 표시도 단순 환율 환산일 뿐 실제 국내 종목 조회 기능 없음
4. **프로그램매매/신용/대차/공매도/CFD 데이터 전무**
5. **뉴스/실적(재무제표) API 미연동** — 플레이스홀더 텍스트만 존재
6. **골든크로스/데드크로스 등 매수·매도 신호 감지 로직 없음** (이동평균 계산만 있고 이벤트 감지는 없음)
7. `generateDemoData()` 등 안 쓰는 죽은 코드 존재

## 4. 데이터 현실성 검토 결과 (중요 — 기대치 조정 완료, 2026-07-14 갱신)

| 항목 | 결정된 처리 방식 |
|---|---|
| **국내+해외주식 시세/차트(OHLC)** | ✅ **토스증권 Open API로 완전 통합** (KIS 안 씀, Yahoo도 개별 종목 조회용으로는 안 씀). AAPL 등 미국 주식도 `/api/v1/prices`, `/candles`로 정상 조회 확인됨(문서에 "국내 및 미국 주식" 명시돼 있었음). 부가로 주문 실행/조건주문(자동매매) API도 제공 — Phase 3에서 알림 대신 실매매 자동화 확장 가능 |
| **환율(USD/KRW)** | ✅ 토스 `/api/v1/exchange-rate`로 통합 (Yahoo KRW=X 제거) |
| **매크로 지표(VIX, 미국 10년물, BTC)** | ⚠️ 토스는 증권사 API라 지수/암호화폐를 종목으로 취급 안 함(VIX·BTC-USD 조회 시 빈 배열, market-indicators는 국내 지수·국채 전용) → **Yahoo Finance 계속 사용** |
| **프로그램매매, 공매도, 대차잔고, 신용잔고 (4종)** | ✅ **KIS Developers API로 최종 확정** (2026-07-15 재검토). KRX 정보데이터시스템은 리뉴얼된 "Data Marketplace"에서 로그인 필요로 막혀있음을 확인 → 대신 KIS 공식 문서에서 4종 전부 종목별/일별 전용 엔드포인트를 찾음: `GET /uapi/domestic-stock/v1/quotations/program-trade-by-stock-daily`(프로그램매매), `daily-short-sale`(공매도), `daily-credit-balance`(신용잔고), `daily-loan-trans`(대차거래). 전부 종목코드+조회기간 파라미터로 일자별 시계열 응답(과거 데이터 백필도 가능). 공매도 엔드포인트는 실전 TR ID(FHPST04830000)까지 확인 완료, 모의투자 미지원(실전 계좌 전용). D+1 공시 데이터라 **"전일 데이터 기반 해석 + 내일 예상" 방식**은 그대로 유지 |
| **CFD 수급** | ❌ **포기하기로 결정** (국내 CFD 취급 증권사 소수, 데이터 API 공개 안 됨) |
| **국내 재무제표(실적)** | DART(전자공시시스템) Open API — 무료 |
| **해외 재무제표/실적 컨센서스** | 기존 Yahoo Finance 활용 (무료 대안은 정확도 낮음 — 필요시 유료 데이터 검토) |

> KIS Developers 재도입 확정 — 시세/차트/환율은 계속 토스증권 API 사용, **수급 4종(프로그램매매/공매도/대차/신용잔고)은 KIS Developers API 전용**으로 병행. KRX 정보데이터시스템 직접 스크레이핑은 로그인 필요 이슈로 포기.

## 5. 확정된 아키텍처

```
[프론트엔드] RAVEN 웹앱 (기존 HTML/CSS/JS 개선)
        ↕ (API 호출)
[백엔드 서버] Node.js — 신규 구축
        ├─ 토스증권 Open API     → 국내+해외 개별 종목 시세/차트(OHLC), 환율(USD/KRW), 추후 주문/자동매매 확장 가능
        ├─ KIS Developers API    → 프로그램매매/공매도/대차/신용잔고 4종 (종목별·일별, 매일 새벽 자동수집)
        ├─ Yahoo Finance         → 매크로 지표 전용(VIX/미국 10년물/BTC — 토스가 지수·암호화폐 미지원). corsproxy.io는 완전 제거, 서버가 직접 호출
        ├─ DART Open API         → 국내 실적(분기 매출/영업이익)
        ├─ 뉴스 API              → 국내(네이버뉴스)/해외(NewsAPI 등)
        ├─ Claude API (Anthropic) → 실제 AI 서술 분석 (API 키는 서버에만 보관, 프론트 노출 금지)
        └─ DB (Supabase)         → 관심종목, 알림설정, 수급데이터 캐시
        ↕
[스케줄러] 정해진 시간마다 자동 실행 (예: 새벽 6시 수급데이터 수집, 5분마다 관심종목 체크)
        ↕
[알림] 조건 충족 시 → 텔레그램 봇으로 메시지 발송
```

### 확정된 기술 스택
| 영역 | 선택 | 이유 |
|---|---|---|
| 백엔드 | Node.js + Express | JS로 프론트/백엔드 통일, 초심자가 새 언어 안 배워도 됨 |
| 배포 | Render.com | 무료 티어, 24시간 상시 실행(실시간 감시 필수), 문서 친화적 |
| DB | Supabase (Postgres) | 대시보드 클릭 기반 관리, 무료 티어 넉넉함 |
| 실시간 알림 | 텔레그램 봇 | 이메일/웹푸시보다 구현 압도적으로 쉬움 |
| AI 분석 | Claude API | 서버에서 지표 요약 → 자연어 해석 생성 |
| 국내 시세/차트 | 토스증권 Open API | 사용자 증권계좌로 발급 완료 (Client Id/Secret 확보) |
| 국내 수급데이터 | KIS Developers API | 프로그램매매/공매도/대차/신용잔고 4종 전용 엔드포인트 확인, 가입 필요 |

> ⚠️ Yahoo `quoteSummary`(회사 프로필/섹터) 엔드포인트는 Yahoo 측에서 크럼/쿠키 인증을 요구하도록 바뀌어 401 발생 — chart 엔드포인트는 정상. 원래도 실패 시 null 처리되던 부분이라 앱 동작에는 영향 없음. 필요 시 별도 해결 필요. 국내 종목은 애초에 프로필 조회를 스킵하도록 처리함.
> ⚠️ 국내 종목 테스트 중 발견한 통화 표시 버그 수정함 — 원화 가격을 달러 포맷터에 넣고 환율을 또 곱해 완전히 잘못된 숫자가 나오던 문제. `formatPrice()`(국내=₩, 해외=$) 도입으로 해결.
> ✅ 포지션 사이즈 계산기 완료 — 원래 CALC 버튼에 로직이 전혀 연결 안 되어 있었음(발견 후 구현). 국내(₩)/해외($) 통화 자동 전환, 진입가 자동입력 지원.
> ✅ TradingView 국내 종목 에러 처리 완료 — `KRX:005930` 형식이 맞는데도(카카오 등 다른 종목으로도 재현) 무료 위젯이 KRX 실시간 데이터 재배포를 막아둔 것으로 확인(라이선스 제약, 심볼 포맷 문제 아님). 위젯 호출 자체를 생략하고 "차트 준비 중" 안내로 대체. 결과 화면 타이틀은 국내/해외 모두 종목명만 표시(코드/티커 없이 — 예: "삼성전자", "애플") 하도록 최종 확정됨.
> 📌 후속 과제(별도 트래킹): 국내 종목 자체 차트(`lightweight-charts` + 토스 캔들 데이터) — Phase 2 이후 후순위.

## 8. 작업 시 주의사항

- 사용자는 코딩 초심자 → 새로운 개념/명령어는 왜 필요한지, 무엇을 하는지 쉽게 설명하고 진행
- API 키, PIN 등 민감정보는 절대 프론트엔드(클라이언트 JS)에 노출하지 말 것 — 반드시 서버 사이드에서만 처리
- 기존 코드의 디자인/UX(PIN-인트로-엔트리 흐름, 스모크 배경 애니메이션 등)는 유지하면서 기능만 확장하는 방향
- CFD 관련 기능은 요청 범위에서 제외됨 (구현 시도 불필요)
